## Objectives ##
Graph learning aims to infer the underlying graph structure from observed data associated with a set of vertices. Assuming that each vertex is associated with $n$ observed values or signals, the objective is to recover the edge set that best represents the relationships between vertices. A major challenge in graph learning is the ill-posed nature of the problem, since many different graph structures may explain the same observations.

## How to define an edge between two nodes?

Given a data matrix $\mathbf{X} = { \mathbf{x}_1, \mathbf{x}_2, \odots, \mathbf{x}_n } \in \mathbb{R}^{p \times n}$ containing $n$ observations, where each observation $\mathbf{x}_i = [x_1, x_2, \odots, x_p]$ ($i \in [1,n]$). 

Several definitions of the relationship between two vertices can be considered. A first approach defines edges according to the distance between the geometric embeddings of the nodes, where nearby embeddings indicate stronger relationships. More specifically, if the observations associated with each node are represented by an embedding vector, an edge can be established between two nodes $x_q$ and $x_l$ based on the distance between their embeddings $\mathbf{X}_q$ and $\mathbf{X}_l$, where $\mathbf{X}_{p}$ is the row vector of data matrix $\mathbf{X}$:

$$
d(\mathbf{X}_q,\mathbf{X}_l)
=
\|\mathbf{X}_q-\mathbf{X}_l\|_2 .
$$

Another approach relies on conditional statistical dependencies between nodes under a given probabilistic model. In this setting, two nodes $x_1$ and $x_2$ are connected if
$$
P(x_1 \mid x_2, \text{other nodes}) \neq P(x_1 \mid \text{other nodes})
$$

which means that knowing $x_2$ provides additional information about $x_1$, even after accounting for all the other nodes. Therefore, an edge is introduced between $x_1$ and $x_2$.
