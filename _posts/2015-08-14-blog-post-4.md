## Definition of graphs ##

### Graph

$$
\mathcal{G}=(\mathcal{V},\mathcal{E},\omega)
$$

- Nodes (vertices): $\mathcal{V}=\\{1,\dots,p\\}$, representing the variables

- Edges:
$\mathcal{E}\subseteq\mathcal{V}\times\mathcal{V}$, representig the relationship between nodes

- Weights: $\omega:\mathcal{E}\to\mathbb{R}$, for $(q,l)\in\mathcal{E}, \ \ w_{q,l}=\omega((q,l)), \ \ \mathbf{w}\in\mathbb{R}^{p(p-1)/2}$, representing the strong/weak relationship between nodes
  
### Matrix representations

- **Adjacency matrix**

$$
[\mathbf A]_{q,\ell}=
\begin{cases}
w_{q,\ell}, & (q,\ell)\in\mathcal E,\\
0, & \text{otherwise}
\end{cases}
$$

- **Graph Laplacian**
$$\mathbf{L}=\mathbf D-\mathbf{A},$$ and the degree matrix $$\mathbf{D}=\mathrm{diag}(\mathbf{A}\mathbf{1})$$ is a diagonal matrix whose entries contain the total connectivity (or weighted degree) of each node

## The role of Graphs in data analysis and data science ##

Graphs play an important role in both data analysis and data science. In data analysis, graphs are commonly used as tools for visualization (e.g., NetworkX, Pyvis) and for unsupervised graph clustering algorithms, such as spectral clustering [Ng et al., 2002a], modularity maximization [Newman et al., 2004; Hollocou et al., 2019], random-walk-based approaches [Pons et al., 2005], and label propagation [Cordasco et al., 2011]. Graphs are also widely used in Graph Signal Processing (GSP) for the design of graph filters and spectral representations [Shuman et al., 2013].

In data science, graph structures are leveraged to enrich node embeddings by combining the intrinsic information of each node with the topological information provided by the graph structure. This integration significantly improves the performance of downstream tasks, such as node classification, link prediction [Zhang et al., 2021].

## The objective of graph learning ##
Graph learning aims to infer the underlying graph structure from observed data associated with a set of vertices. Assuming that each vertex is associated with $n$ observed values or signals, the objective is to recover the edge set that best represents the relationships between nodes. A major challenge in graph learning is the ill-posed nature of the problem, since many different graph structures may explain the same observations.

## How to define an edge between two nodes ? ## 

Given a data matrix $\mathbf{X} = \\{ \mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n \\} \in \mathbb{R}^{p \times n},$ containing $n$ observations, where each observation is defined as $\mathbf{x}_i = \[x_1, x_2, \ldots, x_p\]^\top, \ i \in \\{1,\ldots,n\\},\$ $p$ is the number of variables. 

Several definitions of the relationship between two vertices can be considered. A first approach defines edges according to the distance between the geometric embeddings of the nodes, where nearby embeddings indicate stronger relationships. More specifically, if the observations associated with each node are represented by an embedding vector, an edge can be established between two nodes $x_q$ and $x_l$ based on the distance between their corresponding embeddings $\mathbf{X}_q$ and $\mathbf{X}_l$, where $\mathbf{X}_p$ denotes the $p$-th row vector of the data matrix $\mathbf{X}$:

$$
d(\mathbf{X}_q,\mathbf{X}_l)
= \|\mathbf{X}_q-\mathbf{X}_l\|_2.$$

Another approach relies on conditional statistical dependencies between nodes under a given probabilistic model. In this setting, two nodes $x_1$ and $x_2$ are connected if

$$
P(x_1 \mid x_2, \text{other nodes}) \neq P(x_1 \mid \text{other nodes})
$$

which means that knowing $x_2$ provides additional information about $x_1$, even after accounting for all the other nodes. Therefore, an edge is introduced between $x_1$ and $x_2$.


## The framework of Graph Learning ##
In this part, we decompose principle components of a graph learning framework in order to learn an underlying graph structure explaining the data observations $\mathbf{X} \in \mathbb{R}^{n \times p}$, $n$ is number of samples and $p$ is number of variables :

$$
\begin{aligned}
\underset{\boldsymbol{\Theta}}{\underbrace{\text{minimize}}_{\color{pink}\text{Opt. Solver}}}
\quad &
{\underbrace{\mathcal{L}(\boldsymbol{\Theta},\mathbf{X})}_{\color{magenta}\text{data-fitting term}}}
+
{\underbrace{\lambda h(\boldsymbol{\Theta})}_{\color{orange}\text{sparsity-promoting penalty term}}}
\\
\text{subject to}
\quad &
\underbrace{\boldsymbol{\Theta} \in \mathcal{S}_{\boldsymbol{\Theta}}}_{\color{green}\text{structured precision}}
\\
&
\underbrace{\boldsymbol{\Sigma} \in \mathcal{S}_{\boldsymbol{\Sigma}}}_{\color{cyan}\text{structured covariance}}
\text{and} \qquad
\boldsymbol{\Theta}^{+} = \boldsymbol{\Sigma}.
\end{aligned}
$$

+ $\color{magenta}\text{data-fitting term}$: $\mathcal{L}(\boldsymbol{\Theta}, \mathbf{X})$ learns the graph structure directly from the data observations, which can be designed as a robust function to defined problems (missing data, outliers, corrupted data, etc.,), or, the underlying statistical model generating data can be applied, for example, the negative Gaussian log-likelihood function:
  
$$
\mathcal{L}(\mathbf{\Theta},\mathbf{X}) = {\rm tr(\mathbf{S}\mathbf{\Theta})}-{\rm log|\mathbf{\Theta}|}+{\rm const},
$$

<div style="margin-left: 40px;">
 where $\mathbf{S} = \frac{1}{n}\mathbf{X}^{\top}{X}$ is empirical sample covariance. This function sets a basic for various works, notably GLasso [Friedman et al, 2008] and Laplacian-GMRF [Gilmez et al., 2017]. Another option, e.g, Elliptical distribution, can be applied to model more general behaviors of data (heavy-tailed, contaminated by impulsive noise, interupted by significant outliers, etc.,). 
</div>

+ $\color{orange}\text{Sparsity-promoting penalty term}$: $h(\boldsymbol{\Theta})$, which goes with a regularization parameter $\lambda$, is commonly used in any graph learning algorithm, to promoting the sparsity structure in graph, since the underlying graph inferred from data observations is expected to contain zero correlations between nodes, instead of fully connections. Several options of the sparsity-penalty term can be used, notably $\ell_1$-norm, then non-convex penalties (SCAD, MCP, capped-$\ell_1$).
  
+ $\color{green}\text{Structured precision}$, $\boldsymbol{\Theta}$, or $\color{cyan}\text{structured covariance}$, $\boldsymbol{\Sigma}$. In graph learning, if the matrix $\boldsymbol{\Theta}$ is indentified as the precision matrix, $\boldsymbol{\Theta} = \boldsymbol{\Sigma}^{\dagger}$, the support of graph. Here, the precision estimation is a representative of the adjacency matrix and an edge is assigned if any of its coefficient is greater than a defined threshold. Several options of structural constraints can be directly forced onto the precision $\boldsymbol{\Theta}$, including Symmetric Positive Definite (SPD), the total positivity (MTP2), Laplacian, Spectral constraint for eigenvalues of Laplacian precision. Another approach of forcing structural constraint onto the covariance $\boldsymbol{\Sigma}$, equal to $\boldsymbol{\Theta}^{\dagger}$, is exploited by using factor-models
